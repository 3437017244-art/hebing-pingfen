在线部署

使用 GitHub Pages 托管网页。改完运行 deploy.ps1 上传，手机电脑打开同一地址即可。

你的在线地址

电脑、手机、APP 都使用下面这个 GitHub 地址。

GitHub Pages

https://YOUR_GITHUB_USERNAME.github.io/hebing-pingfen/

首次部署

GitHub Pages（第一次，只需做一次）

  1. 注册 GitHub 账号（若已有可跳过）。
  2. 在 GitHub 新建仓库，名称填 hebing-pingfen，选 Public，不要勾选 README。
  3. 在本文件夹打开 PowerShell，运行下面命令（把用户名换成你的）：
cd /d D:\RuanJian\合并评分 git init git branch -M main git add . git commit -m "首次部署在线版" git remote add origin https://github.com/YOUR_GITHUB_USERNAME/hebing-pingfen.git git push -u origin main
  1. 打开 GitHub 仓库 → Settings → Pages → Build and deployment 选 GitHub Actions。
  2. 等 1～2 分钟，访问 GitHub 地址，能看到本页面即成功。

以后每次改完网页

在本文件夹 CMD 或 PowerShell 运行（一条一条执行）：

cd /d D:\RuanJian\合并评分 powershell -ExecutionPolicy Bypass -File .\deploy.ps1

脚本会提交并推送到 GitHub，GitHub Actions 自动发布到 Pages。

数据自动同步（已配置）

商品/商店数据通过 jsonblob 云端同步，同步码已写在 site-config.js 的 defaultSyncCode 中。

正在读取…
  1. 打开网页时会自动从云端拉取并合并本地数据。
  2. 增删改后约 2 秒自动上传到云端，无需点「立即同步」。
  3. 电脑、手机、APP 打开同一 GitHub 地址即可共享数据。
  4. 若要更换同步码：在「云端同步」页生成新码，写入 site-config.js 后重新 deploy。

手机 APP（与电脑保持一致）

APP 通过在线地址加载网页，界面、功能与电脑版始终同一套代码。数据通过 GitHub 云同步互相同步。

  1. 首次安装：在 D:\RuanJian\合并评分-app 运行下面命令，用 Android Studio 打包 APK 安装到手机（只需做一次）。
  2. 以后更新：电脑改完运行 deploy.ps1 上传即可。APP 会自动检测新版本，也可下拉刷新或到「云端同步」页点「检查更新」。
  3. 数据同步：电脑和手机各在「云端同步」页保存一次 GitHub 令牌，登记后约 2 秒自动同步。
cd D:\RuanJian\合并评分-app npm install npm run apply-online-url npx cap sync android npm run android
每次 deploy.ps1 会自动递增版本号并写入 buildId，APP 打开或回到前台时会提示更新。

让手机 APP 加载在线地址

  1. 打开 D:\RuanJian\合并评分-app\online-site.json，把 siteUrl 改成下面的 GitHub 地址。
  2. 在 APP 项目目录运行：
cd D:\RuanJian\合并评分-app npm run apply-online-url npx cap sync android
  1. 用 Android Studio 重新打包 APK 并安装到手机（只需做一次，之后改网页不用再装 APP）。
  2. 数据会随 site-config.js 中的同步码自动同步，无需手动复制同步码。

工作原理

电脑改网页 → deploy.ps1 上传到 GitHub → GitHub Actions 发布 Pages → 浏览器/APP 打开 GitHub 地址即最新版。商品/商店数据由 defaultSyncCode 自动云端同步（与网页托管无关）。
